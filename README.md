# WildFly Tools - Server Adapter for Eclipse 2026-06

This project is a streamlined version of the original [JBoss Tools](https://github.com/jbosstools/jbosstools-server), generated by AI using the **DeepSeek V4 Flash** model. It provides only the essential WildFly server adapter components for Eclipse, removing unnecessary dependencies and simplifying the project structure.

> Original project: [https://github.com/jbosstools/jbosstools-server](https://github.com/jbosstools/jbosstools-server)

## Components

- **Server adapter** for WildFly, JBoss AS, EAP, and WildFly Legacy
- **JMX tools** for local and remote JVM monitoring (via Jolokia)
- **Archive management** for project packaging
- **Runtime core** for runtime detection and configuration
- **Foundation** with common utilities, validation, and EL support

## Prerequisites

- **Java 17+**
- **Maven 3.8+**

## Build

The build uses **Maven + Tycho** and generates a p2 update site compatible with Eclipse.

### All platforms

Tycho compiles for **Linux (GTK x86_64)**, **Windows (Win32 x86_64)**, and **macOS (Cocoa x86_64)** by default, as configured in `pom.xml`:

```bash
mvn clean install
```

### With target platform (recommended)

```bash
mvn clean install -P target-platform
```

### Build offline

```bash
mvn clean install -o
```

### Specific module

```bash
# Single plugin
mvn clean install -pl plugins/base/org.jboss.tools.foundation.core

# Specific feature
mvn clean install -pl features/org.jboss.tools.jmx.feature
```

### Host + test fragment combined build

```bash
# Build both host and test fragment to compile and run tests
mvn clean install -pl plugins/jmx/org.jboss.tools.jmx.jolokia,plugins/jmx/org.jboss.tools.jmx.jolokia.test
```

### Clean rebuild

```bash
mvn clean install
```

### Skip tests (not recommended)

```bash
mvn clean install -DskipTests
```

## Installation in Eclipse

After the build, the update site is available at:

```
site/target/repository/
```

### Method 1 — Install via local Update Site

1. In Eclipse, go to **Help > Install New Software…**
2. Click **Add… > Local…** and select the `site/target/repository/` directory
3. Check **"WildFly Tools"** in the list of available features
4. Proceed and complete the installation
5. Restart Eclipse

### Method 2 — Install via dropins

Copy the JARs generated in `plugins/*/target/*.jar` to the `dropins/` directory of your Eclipse installation and restart.

### After installation

1. Open **Window > Preferences > Server > Runtime Environments**
2. Click **Add…** and select **WildFly**, **JBoss AS**, or **EAP**
3. Point to the server installation directory
4. Open the **Servers** perspective and create a new server

## Importing into Eclipse IDE for development

1. Install **m2e** and **Tycho Configurator** in Eclipse
2. **File > Import > Existing Maven Projects**
3. Select the project root (`wildfly-tools`)
4. Enable the **Target Platform** in **Preferences > Plug-in Development > Target Platform**

## Project structure

```
├── plugins/
│   ├── base/                  # Infrastructure plugins (foundation, common, runtime)
│   ├── as/                    # Server adapter plugins
│   ├── archives/              # Archive management plugins
│   └── jmx/                   # JMX plugins (local, Jolokia, JVM Monitor)
├── features/                  # Eclipse feature definitions
└── site/                      # p2 update site (eclipse-repository)
```
